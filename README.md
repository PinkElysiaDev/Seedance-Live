# Seedance Live · 使用教程

Seedance Live 是一个**纯前端**的 Seedance 2.0 视频生成工作台。调用字节跳动火山方舟的 doubao-seedance-2.0 系列模型，支持文生视频、首尾帧图生视频、参考图/视频/音频、续帧成链、联网搜索与音频生成。

所有数据（任务、素材、生成的视频）都存在浏览器本地（IndexedDB），无后端、无云存储。API Key 仅保存在你自己的浏览器中。

## 1. 快速开始

### 环境要求

- Node.js 18 或更高
- npm（随 Node 安装）

### 安装与启动

```bash
cd seedance-live
npm install
npm run dev
```

终端会显示本地地址，通常是：

```
http://localhost:5173/
```

在浏览器打开即可进入工作台。

> 首次使用前，请先完成 [第 2 节 首次配置](#2-首次配置)，否则无法提交任务。

### 生产构建

```bash
npm run build      # 编译到 dist/
npm run preview    # 本地预览构建产物
```

`dist/` 是纯静态文件，可部署到任意静态托管（Vercel / Cloudflare Pages / GitHub Pages / Nginx 等）。

## 2. 首次配置

点击右上角 **⚙ 设置**，进入「供应商」标签页。

默认已内置一个「火山方舟官方」配置，你需要填入：

| 字段 | 说明 |
|---|---|
| **名称** | 给这个配置起个名字，便于区分 |
| **Base URL** | 火山方舟接入点，默认 `https://ark.cn-beijing.volces.com/api/v3`，一般无需修改 |
| **API Key** | 火山方舟控制台获取的 API Key（以 `Bearer` 方式鉴权） |

填写后点击 **测试连接**。若返回「连接成功（鉴权通过）」即配置完成。

> API Key 从哪里来？登录[火山方舟控制台](https://console.volcengine.com/ark)，开通 Seedance 模型并创建 API Key。

### 关于 CORS（重要）

浏览器直连火山方舟官方接口**大概率会被 CORS 拦截**。若你点「生成」后立刻报网络错误，说明需要配置代理。请看 [第 8 节 代理](#8-代理绕过-cors)。

## 3. 生成第一个视频

回到主界面，输入区从上到下：

1. **提示词**：描述你想生成的视频。建议 500 字以内，硬上限 4000 字。详细描述主体动作、运镜、光影、场景效果更佳。
2. **素材区**：文生视频可留空，见 [第 4 节](#4-进阶图生视频与参考素材) 使用素材。
3. **参数面板**：选择模型、比例、分辨率、时长等。
4. 点击 **生成视频**。

提交后，任务会出现在下方列表，状态从「排队中 → 生成中 → 完成」。生成通常需要数十秒到数分钟。

**实时校验**：输入区下方会即时显示参数错误（如超量、互斥冲突），并阻止提交，直到问题解决。

### 运行中的任务

- 显示已用时秒数与进度条
- 可随时点 **取消** 中止
- 刷新页面后，**已提交的任务会自动恢复轮询**，不会丢失

### 完成后

- 视频自动下载到本地 IndexedDB，可随时回看，不依赖远程 URL 时效
- 点击视频或卡片打开 **详情**，可全屏播放、下载、续帧、复用配置

## 4. 进阶：图生视频与参考素材

素材区有 5 个槽位：

| 槽位 | 数量上限 | 单文件上限 | 用途 |
|---|---|---|---|
| 首帧 | 1 | 30MB | 作为视频第一帧 |
| 尾帧 | 1 | 30MB | 作为视频最后一帧 |
| 参考图 | 9 | 30MB | 风格/主体参考 |
| 参考视频 | 3 | 50MB | 运镜/动作参考 |
| 参考音频 | 3 | 15MB | 配合参考图/视频使用 |

### 两种添加方式

每个槽位都支持：

- **上传 / 拖入**：点击按钮选文件，或直接把文件拖到整个输入区
- **粘贴公网 URL**：填入 `https://...` 或 `asset://` 地址回车

> 本地文件会被转成 base64 提交。图片通常无碍；**视频/音频若接近上限，base64 体积膨胀可能导致请求过大**，此时建议改用公网 URL。

### 互斥规则（系统会自动校验并提示）

- **首尾帧与参考视频/音频不能同时使用**
- 参考音频必须配合参考图或参考视频
- 参考视频单条时长 2–15 秒，总时长 ≤15 秒
- 参考音频总时长 ≤15 秒

### 参考素材编号

当你添加了参考素材，提交时系统会自动在提示词前拼接编号说明，例如：

> 参考素材编号：图片1、视频1。请按这些编号理解提示词中的图片、视频和音频引用。

你可在提示词里用「图片1」「视频1」等指代素材，模型会据此理解。

## 5. 续帧：把视频接长

Seedance 支持「返回末帧」——生成视频的同时返回最后一帧画面。以此帧作为下一个视频的首帧，即可拼接成长片。

### 操作步骤

1. 在参数面板勾选 **返回末帧（续帧）**
2. 生成完成后，任务卡片或详情页会出现 **续帧** 按钮（仅当末帧成功生成时显示）
3. 点击 **续帧**，系统自动：
   - 取该任务末帧作为新任务的首帧
   - 沿用原任务参数（末帧开关继续开启，可继续往下接）
   - 创建新任务并提交

### 查看续帧链

在任务详情中，顶部会显示 **续帧链路**：

```
[第1段] → [第2段] → [第3段] → ...
```

每个节点是该段视频的缩略图与状态。点击任意节点可在详情中切换查看该段，链路完整可视。

> 续帧依赖末帧图像。若生成时未勾选「返回末帧」或末帧下载失败，续帧按钮不会出现。

## 6. 任务管理

### 任务列表

- 按创建时间倒序排列
- 顶部支持 **按状态筛选**（全部 / 生成中 / 完成 / 失败）和 **按提示词搜索**
- 每张任务卡片显示：封面、状态、模型、比例、时长、提示词摘要

### 任务操作

| 操作 | 说明 |
|---|---|
| 点击卡片 | 打开详情（大图播放 + 完整参数 + 续帧链） |
| 下载 | 保存为本地 mp4 文件 |
| 续帧 | 接着此视频继续生成（需有末帧） |
| 复用 | 把该任务的提示词和参数填回输入区，快速再调一次 |
| 重试 | 失败/取消/过期的任务重新提交 |
| 取消 | 中止运行中的任务 |
| 删除 | 删除任务及其关联的视频/素材（不可撤销） |

### 刷新与恢复

- 刷新或重开浏览器，**所有任务历史保留**
- 运行中或可恢复的任务会**自动继续轮询**，无需手动干预
- 仅「已提交到远端」的任务能恢复；提交瞬间断网的任务会标记失败，需手动重试

### 桌面通知

设置中开启「生成完成桌面通知」后，视频生成完成会弹系统通知（需授权浏览器通知权限）。适合提交长任务后切走做别的事。

## 7. 自定义接口（第三方中转）

除官方接口外，可对接任意 Seedance 兼容的第三方中转。自定义接口采用「**请求模板 + 字段路径**」方式配置：你写一段 JSON 请求体模板（用 `{{prompt}}` 等占位符），再填好各字段在响应里的提取路径即可。

### 7.1 AI 辅助填写（推荐）

如果中转商提供了 API 文档，可让 AI 自动生成整份配置，免去手填字段路径的麻烦。

**前置**：先在设置 → 「AI 辅助」标签页配置一个大模型（Base URL / API Key / 模型名），用于解析文档。

**操作**：

1. 设置 → 供应商 → 选中（或新建）一个自定义接口
2. 点 **✨ AI 解析文档**，选择输入方式：
   - **粘贴内容**：直接贴 API 文档文本（endpoint / 请求体 / 响应体 / 状态枚举 / 字段路径）
   - **上传截图**：上传文档截图（支持多张）
   - **文档 URL**：填文档网址（需启用代理才能跨域抓取，失败则改用粘贴或截图）
3. 点 **解析**，AI 会生成提交 URL、请求体模板、轮询配置与所有字段路径，并在下方给出预览
4. 检查预览无误后点 **应用到当前自定义源**，配置即写入当前 profile

> AI 生成的字段名以文档为准。不同中转商字段名差异很大（如图片字段可能是 `images`、`image_refs` 等），请对照文档核对，必要时手动微调请求体模板。

### 7.2 手动配置

设置 → 供应商 → 点 **+ 自定义**，填好 Base URL、API Key、模型名后，展开自定义模板区，按中转商文档逐项配置：

**提交配置**

| 字段 | 说明 |
|---|---|
| 提交 URL | 创建任务接口，可含 `{{baseUrl}}` |
| 提交方法 | POST |
| 提交 Headers | 键值表，值可含 `{{apiKey}}`（默认带 `Authorization: Bearer {{apiKey}}`） |
| 提交 Body 模板 | JSON 文本，用 `{{prompt}}`、`{{images}}` 等占位符拼请求体 |

**轮询配置**

| 字段 | 说明 |
|---|---|
| 轮询 URL | 查询任务接口，用 `{{taskId}}` 占位 |
| 轮询方法 | GET |
| 轮询 Headers | 键值表，值可含 `{{apiKey}}` |

**响应提取**

| 字段 | 说明 | 示例 |
|---|---|---|
| task id 路径 | 提交响应里任务 ID 的字段路径 | `data.task_id` |
| 状态路径 | 轮询响应里状态字段路径 | `data.status` |
| 成功值 | 表示完成的 status 值（逗号分隔） | `completed,succeeded` |
| 失败值 | 表示失败的 status 值 | `failed,cancelled` |
| 视频 URL 路径 | 结果视频地址字段路径 | `data.result.videos[0].url` |
| 末帧 URL 路径 | 续帧末帧地址（可选） | `data.last_frame_url` |
| 错误路径 | 失败原因字段路径（可选） | `data.error.message` |
| 进度路径 | 进度百分比字段路径（可选） | `data.progress` |
| 轮询间隔(秒) | 该接口的轮询间隔 | `5` |

字段路径支持点号与数组下标，如 `data.result.videos[0].url`。

### 7.3 可用模板变量

请求体模板中可用的占位符：

| 变量 | 含义 |
|---|---|
| `{{prompt}}` | 提示词（原始文本） |
| `{{model}}` | 模型名（profile 模型名覆盖） |
| `{{ratio}}` | 比例，如 `16:9` |
| `{{resolution}}` | 分辨率，如 `720p` |
| `{{duration}}` | 时长（秒） |
| `{{seed}}` | 随机种子（可空） |
| `{{generateAudio}}` | 是否生成音频（true/false） |
| `{{watermark}}` | 是否加水印（true/false） |
| `{{returnLastFrame}}` | 是否返回末帧（true/false） |
| `{{webSearch}}` | 是否联网搜索（true/false） |
| `{{images}}` | 全部图片（首尾帧+参考图）数组 |
| `{{firstFrameImages}}` / `{{lastFrameImages}}` | 首/尾帧图片数组 |
| `{{referenceImages}}` / `{{referenceVideos}}` / `{{referenceAudios}}` | 参考图/视频/音频数组 |
| `{{baseUrl}}` / `{{apiKey}}` | Base URL / API Key（多用于 header） |
| `{{taskId}}` | 轮询任务 ID（仅轮询 URL） |

### 7.4 关于图片素材

- 本地上传的图片素材会被转成 base64 data URL 填入模板。**部分中转商的上游只接受可下载的公网 https URL，不接受 data URL**。若你的中转商属此类，请改用「**粘贴公网 URL**」方式添加图片素材（素材区支持填 `https://...` 地址），避免 base64
- 部分模型需在提示词里用 `@Image1` 等显式引用图片，否则参考图不会生效

配置完成后点 **测试连接** 验证。

### 切换接口

供应商下拉框选择任一配置即可切换为当前生效接口。所有配置都会保存，可随时切回。

## 8. 代理：绕过 CORS

浏览器直连火山官方接口通常被 CORS 拦截。若你使用的中转商已开放 CORS，则无需代理；否则需自备一个转发代理。

### 启用代理

1. 设置 → 代理 → 勾选 **启用代理**
2. 填入你部署的代理地址，如 `https://your-proxy.workers.dev`
3. 选择改写模式：
   - **query**（默认）：`代理地址?target=原始URL`
   - **path**：`代理地址/原始URL`
4. 保存后回到「供应商」点 **测试连接** 验证

### 部署代理

项目提供示例脚本，见 [`docs/proxy.md`](docs/proxy.md)，含 **Cloudflare Workers** 与 **Vercel** 两种实现，复制即可部署。

> **安全提示**：代理会转发你的 API Key。请只用自己部署的代理，不要用来源不明的公共代理。建议在代理中限制 `target` 仅允许火山方舟域名。

## 9. 数据备份与迁移

设置 → 通用 提供：

- **导出数据**：下载一个 JSON 文件，包含所有任务、素材、视频、封面
- **导入数据**：上传 JSON 恢复，已存在的任务会跳过
- **清空所有任务历史**：删除全部任务及关联文件（不可撤销）

适合换设备迁移、或定期备份。

> 导入的任务若原本处于「运行中」状态，会被标记为失败（无法恢复远端轮询），可手动重试。

## 10. 模型与参数说明

### 可用模型

| 模型 | 最大分辨率 | 特点 |
|---|---|---|
| Seedance 2.0 | 1080p | 标准版，质量最高 |
| Seedance 2.0 Fast | 720p | 快速版，速度优先 |
| Seedance 2.0 Face | 1080p | 人脸模型，不支持 `asset://` |
| Seedance 2.0 Fast Face | 720p | 人脸快速版 |

### 参数

| 参数 | 范围 | 说明 |
|---|---|---|
| 比例 | 16:9 / 9:16 / 1:1 / 4:3 / 3:4 / 21:9 / adaptive | adaptive 仅非 face 模型可用 |
| 分辨率 | 480p / 720p / 1080p | 1080p 仅非 fast 模型可用 |
| 时长 | 4–15 秒 | 滑块调节 |
| Seed | 整数或随机 | 相同 seed 近似复现，🎲 随机 |
| 生成音频 | 开/关 | 是否生成配套音频 |
| 返回末帧 | 开/关 | 续帧所需，见 [第 5 节](#5-续帧把视频接长) |
| 联网搜索 | 开/关 | 启用 web_search 工具 |
| 水印 | 开/关 | 是否加水印 |

## 12. 常见问题

**Q：点「生成」立刻报网络错误？**
A：多半是 CORS。配置代理见 [第 8 节](#8-代理绕过-cors)；或改用支持 CORS 的第三方中转见 [第 7 节](#7-自定义接口第三方中转)。

**Q：刷新页面后任务还在吗？**
A：在。所有任务存 IndexedDB，运行中的任务会自动恢复轮询。

**Q：视频下载后打不开 / 是空的？**
A：可能是远程视频 URL 因 CORS 下载失败，仅保留了临时远程链接。尽快从详情页下载；若远程链接已过期则无法恢复，需重新生成。

**Q：续帧按钮为什么灰色 / 不出现？**
A：续帧需要「返回末帧」生成的末帧图像。确认生成时勾选了该选项，且末帧下载成功。

**Q：API Key 安全吗？**
A：纯前端，Key 只存在你本地浏览器，不上传任何服务器。但任何能访问你浏览器的人都能看到，请勿在公共设备使用。代理模式下 Key 会经代理转发，请用自建代理。

**Q：参考视频/音频提示请求过大？**
A：本地大文件转 base64 体积膨胀约 33%。改用公网 URL 添加，或压缩后再上传。

**Q：可以同时生成多个视频吗？**
A：可以连续提交多个任务，它们会并行轮询。但当前有任务运行中时，生成按钮会暂时禁用，待任一完成后可继续提交。

## 项目结构

```
seedance-live/
├─ src/
│  ├─ components/    # Vue 组件（Composer/Gallery/Settings/common）
│  ├─ stores/        # Pinia 状态（settings/tasks/composer/toast）
│  ├─ provider/      # 视频生成供应商接口（官方/自定义）
│  ├─ net/           # HTTP 客户端与代理
│  ├─ db/            # IndexedDB 封装
│  ├─ lib/           # 工具函数（素材/视频/校验等）
│  ├─ config/        # 模型元数据与参数常量
│  ├─ composables/   # Vue 组合式函数
│  └─ types.ts       # 领域类型
├─ docs/proxy.md     # 代理部署示例
└─ ...
```

如需了解架构设计与实现细节，参见方案文档。
